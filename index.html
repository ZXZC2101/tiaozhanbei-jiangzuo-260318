材料要求

立项申报书 + 创业计划书 + 20页以内PPT + 支撑材料

我们的赛道：科技创新和未来产业

我们不是在学比赛知识，而是在赶节点；校赛最核心的材料就是那20页PPT。

模块一评委最想听什么

1
真实问题

为什么比现有方案更好？
3

记住四个问题：为什么做 · 做什么 · 凭什么能做成 · 做成后有什么价值

模块二PPT的真正任务

PPT的核心任务：在几分钟内让评委快速看懂并相信你的项目

  • 能打的PPT：逻辑顺 > 视觉好看
  • 评委最吃：结论明确、数字扎实、页面克制、重点一眼抓住
一页只说一件事 —— 新手最容易做成"文字墙"或"海报墙"，信息塞满但重点全无。

先把PPT拆成"页级任务书"

封面1页 目录1页 背景与需求2–3页 项目概述与方案3–4页 市场分析与商业模式3–4页 团队介绍2页 项目进展与成果2–3页 社会效益与风险2页 未来规划1–2页 致谢1页
PPT分两层：主讲页（5–7分钟答辩用）+ 备份页（Q&A时随时调出来回答评委提问）

不是让AI一步做完，而是让AI帮你把每一步做对

Step 1 · 把AI"喂对"

先建一个"AI投喂包"，把以下信息整理成文档：

  • 项目简介与定位
  • 目标用户画像
  • 核心痛点与需求
  • 技术方案概述
  • 已有成果 / 原型
  • 团队分工与导师
  • 市场信息 / 竞品
  • 风险点与应对
  • 亮点 / 创新点
  • 风格 / 页数 / 16:9 / 受众

不是"帮我做个PPT"，而是结构化输入——你给的信息越准，AI输出越靠谱。

没有的数据写"待补充"，不要让AI脑补假数据

Step 1 · 参考Prompt（页级规划）

将下方Prompt复制，把项目资料粘贴到最后一行即可

你现在是一名熟悉"挑战杯"大学生创业计划竞赛答辩逻辑的PPT策划顾问。

我现在要参加 2026 年"挑战杯"大学生创业计划竞赛校赛，
赛道为：科技创新和未来产业。
受众为：校赛评委老师。
使用场景为：项目答辩 / 项目展示。
PPT比例为：16:9。
页数要求为：18-20页以内。
整体风格要求为：科技感、简洁、清晰、适合答辩，不要花哨。
表达目标为：让评委快速看懂项目价值、创新性、可行性和团队匹配度。

请根据我接下来提供的项目资料，为我生成一套适用于"挑战杯"校赛的PPT页级规划。
要求严格按照下面的大纲来展开：

1. 封面（1页）：简洁明了，突出项目核心关键词
2. 目录（1页）：列出5-6个核心章节，清晰梳理逻辑
3. 项目背景与需求分析（2-3页）：用数据、案例体现需求必要性
4. 项目概述与解决方案（3-4页）：明确定位、业务与方案，突出创新点
5. 市场分析与商业模式（3-4页）：呈现市场潜力与可持续盈利模式
6. 团队介绍（2页）：匹配项目需求，展示成员与导师优势
7. 项目进展与成果（2-3页）：用实际成果证明项目可行性
8. 社会效益与风险应对（2页）：体现综合价值，给出风险解决措施
9. 未来规划与融资需求（1-2页）：制定务实目标，明确融资相关（若有）
10. 致谢（1页）：礼貌感谢评委

请注意：
1. 不要直接生成PPT文件，只输出"每一页的内容规划"。
2. 输出格式：第X页 / 页面标题 / 本页核心结论（一句话）/ 正文要点（3-5条）/ 建议图表 / 主讲提示
3. 页面标题尽量写成"结论式标题"
4. 没有的数据请标注【待补充】，不要自行编造。

以下是我的项目资料：
【在这里粘贴你的项目简介、技术方案、应用场景、目标用户、市场分析、已有成果、团队信息等】

Step 2 · 先出"脚本"，再谈设计

  • 正确顺序：逐页内容脚本 → 设计，不要一上来就让AI做整套
  • 标题写成结论句，不要写"市场分析"这种目录词
  • 正文要点控制在 3–5 条，短句为主，不要长段落
  • AI生成后先检查：内容重复？逻辑跳跃？空话套话？
AI生成的是初稿，不是定稿；真正拉开差距的是你补进去的真实数据

Step 3 · 让AI生成SVG代码

  • 一页一页生成，不要一次生成整套20页
  • 尺寸：1920×1080，16:9
  • 字体用兼容写法：Microsoft YaHei / PingFang SC / Arial
  • 图表用 SVG 基础元素绘制，没有图片时做占位区域

新手四大坑

⚠ 代码被截断
⚠ 文字溢出
⚠ 元素错位
⚠ 特效导入失败
SVG不是越炫越好，越简单越容易导入成功，先求稳

Step 3 · 参考Prompt（SVG生成）

将下方Prompt复制，把本页内容填入最后的占位区域即可

你现在是一名专业的信息可视化设计师和SVG工程师。

我已经完成了一页"挑战杯"校赛答辩PPT的内容规划，
现在请你把这一页转换为"可直接保存为 .svg 文件"的完整SVG代码，
用于后续导入 PowerPoint 制作答辩PPT。

请严格遵守以下要求：
【页面基本要求】
1. 画布尺寸固定为 1920x1080，16:9 比例。
2. 输出内容必须是完整、独立、可直接保存为 .svg 文件的代码。
3. 不要输出解释说明，不要输出Markdown，只输出纯SVG代码。
4. 页面风格：科技感、简洁、稳重，适合高校竞赛答辩。
5. 主色调：蓝白/深蓝+浅灰，整体清爽，不要花哨。
6. 布局要求清晰，注意留白，不要堆满。
7. 只使用基础图形、文本、分割线、色块，不要使用复杂滤镜、渐变特效、外部CSS/JS/字体链接。
8. 字体：Microsoft YaHei, PingFang SC, Noto Sans SC, Arial, sans-serif

【内容排版要求】
1. 页面顶部放主标题。
2. 核心结论要醒目，单独做强调条或高亮模块。
3. 正文要点用简洁短句，控制可读性。
4. 图表用SVG基础元素绘制简洁版本。
5. 需要图片的地方，预留带边框矩形占位区，框内注明"图片预留区 / 可替换为产品图/场景图/团队照片"。

【兼容性要求】
1. 所有文字必须可见，避免超出边界。
2. 字号不要过小，保证投影和答辩时可读。
3. 元素不要过于复杂，避免导入PowerPoint后错位。
4. 所有模块之间保持统一边距和对齐。

Step 4 · 将SVG导入PowerPoint

  • 命名规范：01_cover.svg / 02_menu.svg / 03_background_1.svg
  • 导入方式：插入 → 图片 → 此设备（不要截图成PNG）
  • 导入后逐页检查：文字换行、颜色偏差、图表错位、页边距统一
  • Microsoft 365 可右键尝试转换为形状进一步编辑
  • 备用方案：提前准备 PNG 兜底版本，防止现场翻车
导入PPT是半成品落地，最后的人工校正才决定它像不像比赛级PPT

最后检查 · 从"能看"到"能打"

  • ✓逻辑检查：每页有结论、章节顺畅、无重复内容
  • ✓数据检查：数字 / 案例 / 成果与申报书一致
  • ✓视觉检查：标题 / 字号 / 颜色 / 图表 / 边距统一
  • ✓答辩检查：重点页能顺畅讲完，有 3–5 页备份页
  • ✓设备检查：投影屏或另一台电脑上试过一遍
好PPT不是最花的那套，而是评委听完能立刻复述：你做什么、为什么创新、为什么能做成

三句话说清分工：摘要"把全篇讲短" / 正文"把项目讲全" / 附件"把你说的话坐实"

先搭结构，再补证据，最后再修语言

各模块怎么套、怎么"吹"得更像样

  • 背景：政策/趋势 + 现实场景 + 痛点数据，不要只写"意义重大"
  • 方案：问题 → 解决 → 运行 → 比现有方案强在哪
  • 创新点：相较于现有方案，在___上做了___改进，因此带来___提升
  • 团队：突出谁负责技术/产品/市场，导师提供什么支撑
  • 成果：已完成什么、验证了什么、还差什么
不能编专利、不能编用户量、不能编融资、不能编测试结果
造假是比赛红线，查到直接取消资格。

申报书与PPT互相喂素材

  • 同一项目的核心表述必须统一：项目定位、痛点、创新点、成果、团队
  • 最后三对照：申报书 ↔ PPT ↔ 答辩稿 ↔ 附件材料
真正拉低可信度的，是前后表述不一致
